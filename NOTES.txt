    The Triton compilation failure originated from these two lines inside
    __moe_matmul_ogs_maxT_kernel

        subscript_1 = orig_rows[v_3]      # boolean / tensor indexing
        subscript_2 = acc[v_3, :]         # boolean / tensor indexing

    tl.tensor does not support NumPy–style boolean indexing, so the
    interpreter tried to execute those expressions eagerly, outside of the JIT
    builder, and raised

        ValueError: Did you forget to add @triton.jit ? (`_builder` argument must be
        provided outside of JIT functions.)

    Fix (minimal ­– no algorithmic changes)

        1. Replace the boolean-indexing with ordinary arithmetic plus the
           already-available row mask.  You can rely on `tl.store`’s `mask=…`
           argument to suppress writes for the padding rows.

        # valid rows inside the current 16-row block
        row_valid = mask_1 & v_3                      # [BS1]  (tl.int1)

        # 16×16 block of output values
        v_12 = acc.to(tl.float16)

        # write everything, but only where the row is valid
        tl.store(
            C + (orig_rows[:, None] * 256 + indices_2[None, :]),
            v_12,
            mask=row_valid[:, None],                  # broadcast over the N dimension
        )

        1. Delete the two offending lines as well as the now-unused reshapes:

        - subscript_1 = orig_rows[v_3]
        - orig_rows_valid = tl.reshape(subscript_1, [_BLOCK_SIZE_1])
        - subscript_2 = acc[v_3, :]
        - v_12 = subscript_2.to(tl.float16)
        - tl.store(C + (orig_rows_valid[:, None] * 256 + indices_2[None, :] * 1),
        -          v_12, mask_1[:, None])

        + row_valid = mask_1 & v_3
        + v_12 = acc.to(tl.float16)
        + tl.store(C + (orig_rows[:, None] * 256 + indices_2[None, :]),
        +          v_12, mask=row_valid[:, None])

    That is enough to let Triton build the kernel successfully; the rest of the
    Python code keeps working unchanged.

    Key points
    •  Triton only understands scalar predicates in if …:; masks for bulk
       operations must be passed through tl.store(…, mask=…).
    •  Boolean / fancy indexing (tensor[mask]) is not implemented – use masks
       instead.
    •  tl.reshape(x, [-1]) (or .reshape([])) converts a 1-element tensor into a
       scalar when you really need to use it in a Python‐level if.
